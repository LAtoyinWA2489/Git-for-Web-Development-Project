37. Walk me through Lambda's git flow.  

      Lambda's fit flow is as followed
      1) Fork project from github and copy SSH link
      2) Open terminal to clone link to local machine
      3) cd into freshly cloned repo
      4) Use "git checkout -b" to make a branch of repo
      5) Open and edit repo
      6) Once at a suitable stopping point, save progress of repo then use "git add ." to secondarily save progress before adding to github
      7) Use "git commit -m 'message' " to commit changes to head but not remote repo just yet. A commit message is manditory.
      8) When ready, use the "git push/-u origin <branchName>" to send changes of your branch to remote repo. 
      9) In GitHub, when commit appears, click "compare & pull request".
      10) On the pull request page, select locoal repo as the base to be compared against the "master/main" repo
      11) Create your pull request
      12) Submit URL to Canvas

      An inline element like <span> or <br> is used inside the line of code and only takes up as much space as necessary.
      A block element like <div> is a block-level element which means that it takes up an entire width of their container.

    5. In your own words, explain the box model. What is the "fix" for the box model, in other words, how do we make all elements respect the width we've given them? 

      The box model is the structure by which a element is visible on a webpage; an element's box model consists of (inward to outermost) the content container that holds the main portion of the element, the  padding which is the space between the content and the border, the border which is the layer right outside the padding and what most people see as the "edge" of an element, and the margin which is the space between the border and the outside of the the element.
